# Checklist de Lanzamiento Farma POS

## 1. Producto

- [ ] Probar login con licencia activa y empresa asociada.
- [ ] Probar usuarios por empresa: `admin_empresa`, `supervisor`, `cajero`.
- [ ] Probar ventas, historico, retiros y cierres.
- [ ] Probar inventario con carga manual e importacion.
- [ ] Probar configuracion de perfil y logo por empresa.
- [ ] Probar ticket impreso con datos del negocio.
- [ ] Probar creacion y renovacion de licencias quincenales, mensuales y anuales.

## 2. Infraestructura

- [ ] Desplegar `support-api` en servidor online.
- [ ] Ejecutar esquema SQL del soporte online.
- [ ] Configurar `supportApiUrl` y `supportApiKey`.
- [ ] Confirmar acceso remoto a soporte con internet.
- [ ] Definir backup de base central.
- [ ] Definir backup de bases por empresa.

## 3. Instalacion

- [ ] Generar instalador final.
- [ ] Probar instalacion limpia en otro PC.
- [ ] Validar activacion de licencia en primer equipo.
- [ ] Validar reasignacion o liberacion de equipo.
- [ ] Confirmar que la app abre sin errores despues de reiniciar.

## 4. Comercial

- [ ] Definir precios finales por plan.
- [ ] Definir precios por numero de equipos.
- [ ] Definir politica de renovacion.
- [ ] Definir politica de cambio de equipo.
- [ ] Preparar brochure o PDF comercial.
- [ ] Preparar mensaje corto para WhatsApp.

## 5. Operacion interna

- [ ] Tener formato para registrar empresa nueva.
- [ ] Tener formato para crear usuarios iniciales.
- [ ] Tener flujo para soporte y escalamiento.
- [ ] Tener responsable tecnico.
- [ ] Tener responsable comercial.

## 6. Piloto

- [ ] Activar 1 cliente interno de prueba.
- [ ] Activar 2 o 3 clientes piloto reales.
- [ ] Recoger errores y observaciones.
- [ ] Ajustar antes de venta masiva.

## 7. Minimo para salir a vender

Si quieren empezar esta misma semana, este es el minimo viable:

- licencias por plan funcionando
- instalador estable
- soporte online funcionando
- un cliente piloto operativo
- material comercial basico
- politica clara de precios y renovacion
